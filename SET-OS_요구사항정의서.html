01 제작 목적
배경 — 왜 아카이브가 OS가 되었는가

SET는 타인의 사진에서 추출한 지시문으로 세계를 측정하는 사진 작업이다. 작업의 구조상 사진은 최종 결과물이 아니라 측정 과정의 출력물에 가깝다. 지시문을 입력하고, 촬영하고, 선별하고, 기록하는 전 과정이 작업 자체이며, 그 과정은 온라인에 공개된다.

이 아카이브를 단순 갤러리나 포트폴리오 사이트로 구성하지 않은 이유는 세 가지다.

1 — 측정 장치로서의 인터페이스

SET의 작업 논리는 "관측자도 측정된다"는 전제 위에 있다. 관객이 수동적으로 사진을 보는 것이 아니라, MEASURE를 통해 지시문 언어를 직접 입력하고 검색하면 교집합이 생성된다. 아카이브가 곧 측정 도구여야 했다.

2 — 과정의 가시화

어떤 사진이 CORE(채택)·SUPPORT(보조)·DROP(탈락)되었는지, 어떤 조건이 반복 출현했는지를 데이터로 드러내야 했다. 갤러리 형식은 선별된 결과만 보여주지만 SET는 과정 전체가 작업이다.

3 — 공집합의 설계

SET:Inclusion(자기 자신의 사진에서 지시문을 역추출하는 작업)이 합류함으로써 두 작업의 교집합 지대, 즉 언어와 이미지 사이의 측정 불가능한 오차 자체를 시각화할 공간이 필요해졌다. 이 공간은 단순 뷰어가 아닌 실시간 데이터 터미널 형식으로 설계되었다.

결과적으로 set-os.html은 사진 아카이브이자 측정 인터페이스이자 작업 공정 모니터다. CRT 터미널 미학은 장식이 아니라 "이 시스템은 측정 중이다"라는 작업 맥락의 시각적 번역이다.
